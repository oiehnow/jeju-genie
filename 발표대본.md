## 1. 문제 정의 (0:45)

제주 여행 정보는 **여기저기 흩어져 있습니다.** 관광공사, 블로그, 맛집 앱, 날씨…
사용자는 검색을 열 번 하고도 "그래서 2박 3일에 뭘 하지?"라는 질문의 답을 못 얻습니다.

기술적으로 이건 두 가지 문제입니다.
- 첫째, **신뢰할 수 있는 출처에 근거한 답변** — 즉 LLM 환각을 막는 RAG가 필요합니다.
- 둘째, **데이터 소스가 계속 늘어난다** — 관광 API, 맛집, 날씨가 나중에 붙어야 합니다.
  그래서 "소스를 꽂으면 바로 붙는" 확장 구조가 처음부터 설계에 들어가야 했습니다.

## 2. 솔루션 개요 + 데모 (1:30)

[화면: 배포된 URL 열기 — https://jeju-genie-1030073717906.asia-northeast3.run.app]

지금 데모 배포본은 LLM 키가 아직 안 들어가서 **설계된 폴백 응답**이 나옵니다.
이 부분은 뒤에서 "왜 일부러 이렇게 설계했는지" 설명드리겠습니다.

## 3. 무엇으로 만들었나 — 기술 스택 (1:30)

아키텍처는 크게 세 덩어리입니다.

**백엔드 — FastAPI**
- SSE 스트리밍 챗 엔드포인트, RAG 파이프라인, 그리고 **커넥터 플러그인 구조**를 담았습니다.
- 커넥터는 `backend/app/connectors/`에 **파일 하나 추가하면 데이터 소스가 하나 늘어나는** 구조입니다.
  visitjeju API든 맛집 크롤러든, 코어를 안 건드리고 확장됩니다.
- pytest 7개로 회귀를 잡습니다. 이게 뒤의 CI에서 배포 게이트 역할을 합니다.

**RAG / 벡터 스토어 — Chroma**
- 시드 지식으로 제주 개요·관광지·음식·팁 4개 문서를 넣어 5개 청크로 인덱싱했습니다.
- 핵심 설계: **컬렉션을 임베딩 모델별로 분리**했습니다. `jeju-<provider>-<model>` 네이밍이라,
  임베딩 모델을 바꾸면 다른 컬렉션으로 격리됩니다. 왜 이게 중요한지는 배포 파트에서 말씀드리겠습니다.
- 임베딩은 로컬에선 nomic, OpenAI 키가 들어오면 자동으로 OpenAI 임베딩으로 전환됩니다.

**LLM 계층 — 폴백 설계**
- OpenAI GPT를 1순위로 쓰되, 키가 없으면 로컬 Ollama(qwen3:14b)로,
  그것도 없으면 **"준비 중" 안내를 반환**합니다.
- 즉 **키가 없어도 서비스가 죽지 않고, 페이지·헬스체크·인덱스는 항상 정상**입니다.
  데모 환경에서 폴백 응답이 뜬 게 바로 이 설계 덕분입니다. 장애가 아니라 의도된 동작입니다.

**프론트엔드 — React**
- 말풍선 채팅 UI, 마스코트 슬롯, 그리고 **테마 토큰(theme.css)**으로 색·크기를 변수화했습니다.
  디자인 변경이 CSS 변수 수정만으로 끝나게 만들었습니다.

## 4. 어떻게 배포했나 — 로컬에서 클라우드까지 (1:30)

**로컬은 Docker Compose 한 방입니다.**
`docker compose up -d --build` 하면 localhost:8090에서 백엔드+프론트가 뜨고,
E2E까지 로컬에서 검증한 뒤 클라우드로 올립니다.

**클라우드는 Google Cloud Run입니다.** asia-northeast3(서울) 리전이고, 여기 세 가지 결정이 있습니다.

- **min-instances 0** — 유휴 시 인스턴스가 0으로 내려가 과금이 없습니다.
  해커톤/데모에 딱 맞는 서버리스 비용 구조입니다. 트래픽이 오면 콜드스타트로 깨어납니다.
- **Chroma 인덱스는 GCS에 저장**하고, **서비스가 시작할 때 자동으로 pull** 합니다.
  컨테이너는 stateless로 유지하고, 지식 데이터는 밖으로 뺐습니다.
  덕분에 재배포해도 인덱스를 다시 만들 필요가 없습니다.
- **이미지는 Artifact Registry**에 올리고 Cloud Run이 그걸 당겨서 뜹니다.
  빌드는 항상 `--platform linux/amd64 --provenance=false --sbom=false`로 고정했는데,
  이건 Cloud Run/AR 검증에서 실제로 걸렸던 함정을 잡은 설정입니다.

## 5. MLOps 과정 — CI/CD 파이프라인 (1:30)

여기가 이번 프로젝트의 **핵심**입니다.

레포는 GitHub `oiehnow/jeju-genie`(비공개)이고, **main에 push만 하면 배포가 끝납니다.**

파이프라인은 이렇게 흐릅니다.

```
main push
  → lint / test (pytest 7개)     ← 실패하면 여기서 배포 차단
  → Docker 빌드 (linux/amd64)
  → Artifact Registry push
  → Cloud Run 자동 배포 (새 리비전)
```

몇 가지 강조하고 싶은 점:

- **테스트가 배포 게이트입니다.** lint/test가 깨지면 클라우드로 안 나갑니다.
  즉 깨진 코드가 프로덕션에 도달할 수 없습니다.
- **비밀은 GitHub Secrets로 주입**합니다. `GCP_SA_KEY`, `GCP_PROJECT_ID`를 등록했고,
  서비스 계정에는 `roles/run.admin`만 최소 권한으로 부여했습니다.
- LLM 키도 같은 방식입니다. `gh secret set OPENAI_API_KEY` 한 번이면
  워크플로가 Cloud Run 환경변수로 자동 주입합니다. **코드에 키가 박히지 않습니다.**

그리고 앞에서 예고한 **"임베딩 모델별 컬렉션 분리"**가 여기서 빛을 발합니다.
임베딩 모델을 바꾸면 벡터 차원과 의미 공간이 바뀌므로 **인덱스를 재구축해야 합니다.**
저희는 컬렉션을 모델별로 격리해뒀기 때문에, 모델 교체 시 옛 인덱스와 섞이지 않고
`ingest --reset --push`로 깨끗하게 재구축→GCS 반영이 됩니다.
**데이터 파이프라인과 모델 버전을 함께 관리한다** — 이게 MLOps의 요점이라고 봤습니다.

## 7. 앞으로의 계획 (1:00)

로드맵은 크게 세 갈래입니다.

**첫째, LLM/임베딩 고도화.**
OpenAI 키를 주입하면 GPT 답변 품질과 OpenAI 임베딩 검색 품질로 한 단계 올라갑니다.
파이프라인은 이미 준비돼 있어서 **키 주입 + 인덱스 재구축 한 번**이면 전환됩니다.

**둘째, 실데이터 커넥터 확장.**
지금은 시드 지식 4문서지만, **커넥터 플러그인 구조** 덕분에
visitjeju 관광 API, 맛집, 날씨, 축제 정보를 파일 추가만으로 붙일 수 있습니다.
"살아있는 제주 데이터"로 채우는 게 다음 목표입니다.

**셋째, 운영 성숙도.**
- 데이터 신선도를 위한 **스케줄드 재-ingest**(예: Airflow — 저희 로컬 8080을 이미 Airflow가 쓰고 있습니다),
- 답변 품질 모니터링과 사용자 피드백 루프,
- 그리고 트래픽이 커지면 min-instances 조정으로 콜드스타트를 완화하는 스케일 정책.

정리하면, **지금은 뼈대와 파이프라인이 완성됐고, 앞으로는 데이터와 품질을 이 파이프라인에 흘려보내는 단계**입니다.

## 부록 A. 예상 Q&A (기술 심사위원 대비)

**Q. 지금 답변이 왜 "준비 중"으로 나오나요?**
> 의도된 폴백입니다. LLM 키가 없을 때 서비스를 죽이는 대신 안내를 반환하도록 설계했습니다.
> 페이지·헬스·인덱스는 정상이고, 키만 주입하면 즉시 실답변으로 전환됩니다.

**Q. RAG 환각은 어떻게 줄이나요?**
> Chroma에서 검색된 시드 지식 청크를 컨텍스트로 넣어 근거 기반으로 답하게 합니다.
> 임베딩 모델별 컬렉션 분리로 검색 일관성도 관리합니다.

**Q. 임베딩 모델을 바꾸면 기존 데이터는요?**
> 컬렉션이 `jeju-<provider>-<model>`로 격리돼 있어 섞이지 않습니다.
> `ingest --reset --push`로 새 컬렉션을 만들고 GCS에 반영합니다. 롤백도 컬렉션 단위로 가능합니다.

**Q. 비용 관리는?**
> Cloud Run min-instances 0이라 유휴 과금이 없습니다. 트래픽 시에만 인스턴스가 뜹니다.

**Q. 배포 안정성은? 잘못된 코드가 나가면?**
> CI에서 lint/test가 게이트입니다. pytest가 깨지면 빌드·배포가 차단됩니다.
> Cloud Run은 리비전 단위라 문제가 생기면 이전 리비전으로 즉시 롤백됩니다.

**Q. 확장성 — 새 데이터 소스 붙이는 데 얼마나 걸리나요?**
> 커넥터 파일 하나 추가 + `ingest`면 됩니다. 코어 코드는 건드리지 않습니다.
